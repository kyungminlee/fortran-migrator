## Testing model

Each driver runs the same operation through (a) the migrated
extended-precision routine and (b) a quad-precision
(`REAL(KIND=16)`) reference built from vendored Netlib sources with
gfortran's `-freal-8-real-16` promotion. Both sides receive identical
inputs (generated by a seeded RNG); the migrated output is compared to
the reference in `REAL(KIND=16)` space and converted to **decimal
digits of agreement** (`-log10(max_rel_err)`).

Pass criterion:
- **kind16** (BLAS + LAPACK): bit-exact (`max_rel_err == 0`) **or**
  `digits ≥ 28`. Quad-precision arithmetic gives ~33 digits, so 28
  leaves ~5-ULP headroom for intrinsic-call differences (sqrt,
  divisions) and the ULPs accumulated by self-residual computations
  in the eigenvalue / SVD drivers — without ever accepting the kind
  of 1–5 % bug a precision-cast slip-up produces.
- **All other targets**: per-case `max_rel_err ≤ tol`, where
  `tol = safety · n_ops · target_eps`.


## Headline observations

Across every passing case (363 reports, 1 386 cases), here's where
each target lands:

| Target       | Theoretical max digits | Median observed digits |
|--------------|-----------------------:|-----------------------:|
| kind10       |                  ~19   |                  ~18.5 |
| kind16       |                  ~33   |                **exact** (bit-identical) |
| multifloats  |                  ~32   |                  ~31   |

- **kind10** lands within 1–2 digits of the IEEE 80-bit ceiling on
  every routine — exactly as expected for differential precision
  against a quad reference.
- **kind16** is **bit-exact** on every BLAS and LAPACK routine where
  the comparison is direct (the migrated routine and the quad
  reference compute the same algorithm at the same precision). The
  only non-`exact` kind16 entries are the eigenvalue/SVD invariant-
  residual computations (`dsyev`, `zheev`, `dgesvd`, `pdsyev`) and
  Cholesky `dpotrf` at large `n` — all land at 32–33 digits, which
  is the precision floor of self-residual matmul in quad arithmetic.
- **multifloats** (double-double, ~32 digits) consistently agrees to
  ~30–31 digits. The 1-digit shortfall vs. the theoretical ceiling
  is the expected double-double rounding.

## Coverage notes

The test drivers exercise more than just one default code path per
routine. Recent rounds of review (commits `91b40af`, `fab6c39`,
`eeeef97`, `c24363f`) systematically broadened coverage along the
character-flag axis:

- **Triangular routines** (`dtrsv`, `dtrmv`, `dtbmv`, `dtpmv`,
  `dtrsm`, `dtrmm`, `ztrsm`): each driver cycles `(UPLO, TRANS, DIAG)`
  over its size loop, hitting upper / lower / transposed / unit-diag
  paths; SIDE='R' is exercised in `dtrsm`/`dtrmm`/`ztrsm`.
- **Symmetric / Hermitian routines** (`dsymv`, `dsymm`, `dsyrk`,
  `zhemv`, `zhemm`, `zherk`): cycle `(UPLO, SIDE, TRANS)`; each
  Hermitian driver also exercises `TRANS='C'` (conjugate-transpose).
- **General complex** (`zgemm`, `zgemv`): cycle `(TRANSA, TRANSB)`
  through NN, CN, NT, CC — the `'C'` path is the most bug-prone in
  complex BLAS.
- **Quick-return paths**: `dgemm` and `pdgemm` each include
  `(α=0, β≠0)` and `(α≠0, β=0)` cases to exercise the branches that
  bypass the inner kernel entirely.
- **Edge cases**: `idamax` has an explicit-plateau case for the
  first-index tie-breaking spec; `drotmg` has 9 cases covering
  `d1<0`, `d2=0`, `y1=0`, and the GAMSQ / 1/GAMSQ rescale branches;
  `dtbmv` includes a `k=0` (diagonal-only) edge.
- **Rectangular / partial-block** shapes: `dgeqrf` includes a wide
  `m=32, n=48` case; `pdgemm` includes an `n=70 ≢ 0 mod mb` case for
  block-cyclic edge testing.
- **JOB-flag coverage**: `dsyev` cycles `(JOBZ, UPLO)` including
  `JOBZ='N'`; ScaLAPACK `pdsyev` cycles `JOBZ='V'`/`'N'` and
  validates eigenvectors via the sign-immune residual
  `‖A·Z − Z·diag(W)‖/‖A‖`.

Sub-percent migration regressions (the kind that produced the
historical PB_Cplascal bug) are caught by the kind16 strict floor;
single-corner-of-the-flag-space bugs are caught by the cycling
above.

### Per-target × per-library coverage

A given test driver runs against any target whose
``target_<TARGET>/target_<library>.f90`` wrapper exists. Today's
matrix:

| Library     | kind10 | kind16 | multifloats |
|-------------|--------|--------|-------------|
| BLAS        | ✓ 75   | ✓ 75   | ✓ 75        |
| LAPACK      | ✓ 75   | ✓ 75   | ✓ 75        |
| PBLAS       | ✓ 20   | ✓ 20   | ✓ 20        |
| ScaLAPACK   | ✓ 11   | ✓ 11   | ✓ 11        |
| **per-target total** | **181** | **181** | **181** |

`✓ N` means N test drivers run on that target. All three targets
now have full BLAS / LAPACK / PBLAS / ScaLAPACK coverage; the only
remaining hole is complex-side ScaLAPACK (`pz*` / `px*` driver),
where the `target_scalapack` wrapper exposes interfaces (`pxgesv`,
`pxgeqrf`, `pxheev`) but no test exercises them.

Per-target wrapper modules are generated at build time from a single
fypp template per framework (`tests/<fw>/common/target_<fw>_body.fypp`)
plus a thin per-target shim that sets the type / prefix / converter.
A new test driver therefore wires up once per framework — the three
targets pick up the new entry point automatically through the shared
body. The quad↔target conversion lives in
`tests/common/target_<target>/target_conv.f90` (identity for kind16,
intrinsic cast for kind10, double-double split for multifloats).

No complex (`pz*`/`px*`) ScaLAPACK driver is tested today — the
`target_scalapack` template exposes complex wrappers (`pxgesv`,
`pxgeqrf`, `pxheev`) but no test exercises them. Adding a complex
ScaLAPACK test driver would close the largest remaining hole.


## Reproducing this report

The build requires `fypp` on `PATH` (used by CMake to expand the
per-target wrapper templates). `uv sync --dev` installs it as a dev
dependency; `pip install fypp` works too.

```bash
nproc                                # confirm worker count for -j
# → suppose 8

# Stage and run all three targets
for t in kind10 kind16 multifloats; do
    rm -rf /tmp/stg-$t
    PYTHONPATH=src python -m pyengine stage /tmp/stg-$t \
        --target $t \
        --libraries blas blacs lapack ptzblas pbblas pblas scalapack scalapack_c
    cmake -S /tmp/stg-$t -B /tmp/stg-$t/build -DCMAKE_BUILD_TYPE=Release
    cmake --build /tmp/stg-$t/build -j8
    ctest --test-dir /tmp/stg-$t/build --output-on-failure
done

# Aggregate the JSON reports
mkdir -p /tmp/all-reports
cp /tmp/stg-kind10/build/precision_reports/*.json     /tmp/all-reports/
cp /tmp/stg-kind16/build/precision_reports/*.json     /tmp/all-reports/
cp /tmp/stg-multifloats/build/precision_reports/*.json /tmp/all-reports/

python scripts/precision_report.py /tmp/all-reports --out tests/REPORT.md
```

### AddressSanitizer build (multifloats / kind16)

Both targets pass cleanly under ASan. The multifloats invocation
needs three extra options because the upstream multifloats
`localize_symbols.sh` POST_BUILD step (and its hidden-visibility
preset, and fat-LTO) interact badly with ASan's link-time symbol
resolution — disabling all three lets the static archive link cleanly
under instrumentation:

```bash
# multifloats + ASan
rm -rf /tmp/stg-mf-asan && \
PYTHONPATH=src python -m pyengine stage /tmp/stg-mf-asan \
    --target multifloats \
    --libraries blas blacs lapack ptzblas pbblas pblas scalapack scalapack_c
cmake -S /tmp/stg-mf-asan -B /tmp/stg-mf-asan/build \
    -DCMAKE_BUILD_TYPE=RelWithDebInfo \
    -DMULTIFLOATS_USE_LTO=OFF \
    -DMULTIFLOATS_HIDDEN_VISIBILITY=OFF \
    -DMULTIFLOATS_STRIP_SYMBOLS=OFF \
    -DCMAKE_C_FLAGS="-fsanitize=address -fno-omit-frame-pointer" \
    -DCMAKE_CXX_FLAGS="-fsanitize=address -fno-omit-frame-pointer" \
    -DCMAKE_Fortran_FLAGS="-fsanitize=address -fno-omit-frame-pointer" \
    -DCMAKE_EXE_LINKER_FLAGS="-fsanitize=address"
cmake --build /tmp/stg-mf-asan/build -j8
ASAN_OPTIONS="detect_leaks=0:abort_on_error=0" \
    ctest --test-dir /tmp/stg-mf-asan/build --output-on-failure

# kind16 + ASan (no multifloats opts needed)
rm -rf /tmp/stg-k16-asan && \
PYTHONPATH=src python -m pyengine stage /tmp/stg-k16-asan \
    --target kind16 \
    --libraries blas blacs lapack ptzblas pbblas pblas scalapack scalapack_c
cmake -S /tmp/stg-k16-asan -B /tmp/stg-k16-asan/build \
    -DCMAKE_BUILD_TYPE=RelWithDebInfo \
    -DCMAKE_C_FLAGS="-fsanitize=address -fno-omit-frame-pointer" \
    -DCMAKE_CXX_FLAGS="-fsanitize=address -fno-omit-frame-pointer" \
    -DCMAKE_Fortran_FLAGS="-fsanitize=address -fno-omit-frame-pointer" \
    -DCMAKE_EXE_LINKER_FLAGS="-fsanitize=address"
cmake --build /tmp/stg-k16-asan/build -j8
ASAN_OPTIONS="detect_leaks=0:abort_on_error=0" \
    ctest --test-dir /tmp/stg-k16-asan/build --output-on-failure
```

To use a non-default multifloats checkout (e.g. `develop` branch, a
fork, or a local working copy) for any build:

- `-DMULTIFLOATS_GIT_TAG=develop` — switch the FetchContent branch
- `-DMULTIFLOATS_GIT_REPO=<url>` — point at a fork
- `-DMULTIFLOATS_DIR=/path/to/checkout` — skip fetch, use a local tree
